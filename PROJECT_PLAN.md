# แผนการดำเนินงานโปรเจกต์เชิงกลยุทธ์ (Strategic Project Roadmap)
**หัวข้อ:** การจำแนกกลุ่มความเสี่ยงทางการเงินจากพฤติกรรมการใช้จ่ายในแต่ละช่วงวัย
**รายวิชา:** AIE225 คณะวิศวกรรมศาสตร์ มหาวิทยาลัยกรุงเทพ

---

## 📌 1. บทนำและวัตถุประสงค์ (Introduction & Objective)
ท่ามกลางวิกฤตค่าครองชีพที่สูงขึ้น โปรเจกต์นี้มุ่งเน้นการศึกษาว่าพฤติกรรมการจัดการเงินของคนแต่ละ Generation (X, Y, Z) ส่งผลต่อความมั่นคงทางการเงินอย่างไร โดยเป้าหมายสูงสุดคือการสร้างโมเดล Machine Learning ที่สามารถจำแนกและทำนาย "ระดับความเสี่ยงทางการเงิน" ของบุคคล เพื่อเป็นเครื่องมือในการวางแผนและเตือนภัยล่วงหน้า

---

## 🛠 2. ขั้นตอนการดำเนินงานอย่างละเอียด (Detailed Execution Plan)

### Phase 1: การเตรียมข้อมูลและวิศวกรรมฟีเจอร์ (Data Preprocessing & Feature Engineering)
*เป้าหมาย: เปลี่ยนข้อมูลดิบจาก Google Form ให้เป็นชุดข้อมูลที่พร้อมสำหรับการวิเคราะห์เชิงลึก*
1. **Data Integration:** รวบรวมข้อมูล 200 ตัวอย่าง นำเข้าด้วย Pandas และทำความสะอาดข้อมูล (Handling Missing Values/Outliers)
2. **Standardization:** เปลี่ยนชื่อหัวข้อคำถามภาษาไทยให้เป็นตัวแปรภาษาอังกฤษที่สื่อความหมาย (เช่น `income`, `impulse_buying`, `debt_ratio`)
3. **Advanced Feature Engineering:** สร้างฟีเจอร์ใหม่ "อัตราส่วนหนี้สินต่อรายได้" (Debt-to-Income Ratio) โดยแปลงข้อมูลรายได้และหนี้สินจากแบบช่วง (Categorical) ให้เป็นตัวเลข (Numerical) เพื่อให้โมเดลเห็นความสัมพันธ์เชิงปริมาณได้ดีขึ้น
4. **Risk Scoring (The Core Logic):** เนื่องจากแบบสอบถามไม่มีการระบุความเสี่ยงโดยตรง เราจะสร้างระบบให้คะแนนความเสี่ยงรวม (Total Risk Score) เต็ม 9 คะแนน จาก 3 ปัจจัยคือ 1) สภาพคล่อง 2) ประวัติหมุนเงินไม่ทัน และ 3) ภาระหนี้สิน เพื่อแบ่งกลุ่มเป้าหมาย (Target) เป็น 3 ระดับอย่างแม่นยำ:
    - **High Risk (เสี่ยงสูง) [คะแนน 6-9]:** กลุ่มที่ไม่มีเงินสำรองและแบกรับหนี้สินสูง
    - **Medium Risk (เสี่ยงปานกลาง) [คะแนน 3-5]:** กลุ่มเฝ้าระวัง อาจมีเงินสำรองบ้างแต่ยังมีหนี้
    - **Low Risk (เสี่ยงต่ำ) [คะแนน 0-2]:** กลุ่มที่มีความมั่นคงทางการเงินสูง หนี้สินน้อยหรือไม่มี

### Phase 2: การวิเคราะห์ Insight และการแสดงผล (Deep EDA & Visualization)
*เป้าหมาย: ค้นหาความสัมพันธ์ที่ซ่อนอยู่ในข้อมูล เพื่อตอบโจทย์ "ใครคือกลุ่มที่น่ากังวลที่สุด"*
1. **Generation Analysis:** เปรียบเทียบภาระหนี้สินในแต่ละรุ่น (Stacked Bar Chart) เพื่อดูว่า Gen Y แบกรับภาระหนี้บ้าน/รถ มากกว่า Gen อื่นจริงหรือไม่
2. **Behavioral Impact:** วิเคราะห์พฤติกรรม "การซื้อของตามใจ" (Impulse Buying) สัมพันธ์กับระดับความเสี่ยงอย่างไร (Heatmap/Boxplot)
3. **Liquidity Mapping:** ใช้ Swarmplot (Scatter Plot แบบกระจายจุดไม่ให้ทับกัน) วิเคราะห์ความสัมพันธ์ระหว่าง "รายได้" และ "เงินสำรองฉุกเฉิน" แบ่งสีตามระดับความเสี่ยง เพื่อพิสูจน์ว่ารายได้สูงไม่ได้แปลว่าความเสี่ยงต่ำเสมอไป
4. **Coping Strategy:** วิเคราะห์แนวทางรับมือวิกฤตเศรษฐกิจของแต่ละ Gen (Grouped Bar Chart)
5. **Financial Literacy vs. Risk:** ใช้ Boxplot วิเคราะห์ความสัมพันธ์ระหว่าง "ระดับความรู้ทางการเงิน" (จากคำถามข้อ 17) กับ "กลุ่มความเสี่ยง" ที่คำนวณได้ เพื่อดูว่าความรู้ส่งผลต่อความเสี่ยงจริงหรือไม่

### Phase 3: การพัฒนาและทดสอบโมเดล (Machine Learning Pipeline)
*เป้าหมาย: สร้างปัญญาประดิษฐ์ที่จดจำพฤติกรรมความเสี่ยง*
1.  **Model Selection:**
    *   **Logistic Regression:** เพื่อดูความน่าจะเป็นในการตกอยู่ในกลุ่มเสี่ยง
    *   **KNN (K-Nearest Neighbors):** เพื่อจำแนกกลุ่มตามความคล้ายคลึงของพฤติกรรม
2.  **Training & Evaluation:**
    *   แบ่งข้อมูลเป็น Training Set (80%) และ Testing Set (20%)
    *   ฝึกสอนโมเดลทั้งสองด้วยชุดข้อมูล Training
    *   ประเมินประสิทธิภาพของโมเดลด้วย Accuracy, Precision, Recall, และ F1-Score
    *   ใช้ Confusion Matrix เพื่อแสดงผลการจำแนกประเภท
3.  **Hyperparameter Tuning (Optional):**
    *   หากมีเวลา จะทำการปรับ Hyperparameter ด้วย GridSearchCV เพื่อเพิ่มประสิทธิภาพของโมเดล

### Phase 4: การนำไปใช้และสรุปผล (Deployment & Recommendations)
1. **Prediction Module:** เขียนฟังก์ชันรับค่าจำลอง (Mock User Input) เพื่อรันโมเดลทำนายผลความเสี่ยงทันที
2. **Strategic Insights:** สรุปปัจจัยที่มีผลต่อความเสี่ยงมากที่สุด (Feature Importance)
3. **Actionable Recommendations:** ให้ข้อเสนอแนะสำหรับการปรับปรุงพฤติกรรมการเงินแยกตามระดับความเสี่ยง
