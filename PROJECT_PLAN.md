# แผนการดำเนินงานโปรเจกต์เชิงกลยุทธ์ (Strategic Project Roadmap)
**หัวข้อ:** การจำแนกกลุ่มความเสี่ยงทางการเงินจากพฤติกรรมการใช้จ่ายในแต่ละช่วงวัย
**รายวิชา:** AIE225 คณะวิศวกรรมศาสตร์ มหาวิทยาลัยกรุงเทพ

---

## 📌 1. บทนำและวัตถุประสงค์ (Introduction & Objective)
ท่ามกลางวิกฤตค่าครองชีพที่สูงขึ้น โปรเจกต์นี้มุ่งเน้นการศึกษาว่าพฤติกรรมการจัดการเงินของคนแต่ละ Generation (X, Y, Z) ส่งผลต่อความมั่นคงทางการเงินอย่างไร โดยเป้าหมายสูงสุดคือการสร้างโมเดล Machine Learning ที่สามารถจำแนกและทำนาย "ระดับความเสี่ยงทางการเงิน" ของบุคคล เพื่อเป็นเครื่องมือในการวางแผนและเตือนภัยล่วงหน้า

---

## 🛠 2. ขั้นตอนการดำเนินงานอย่างละเอียด (Detailed Execution Plan)

### Phase 1: การเตรียมข้อมูลและวิศวกรรมฟีเจอร์ (Data Preprocessing & Feature Engineering)
*เป้าหมาย: เปลี่ยนข้อมูลดิบจาก Google Form ให้เป็นชุดข้อมูลที่พร้อมสำหรับการวิเคราะห์เชิงลึก*
1. **Data Integration:** รวบรวมข้อมูลตัวอย่าง นำเข้าด้วย Pandas และทำความสะอาดข้อมูล (Handling Missing Values/Outliers)
2. **Standardization:** เปลี่ยนชื่อหัวข้อคำถามภาษาไทยให้เป็นตัวแปรภาษาอังกฤษที่สื่อความหมาย (เช่น `income`, `impulse_buying`, `debt_ratio`)
3. **Advanced Feature Engineering:** สร้างฟีเจอร์ใหม่ "อัตราส่วนหนี้สินต่อรายได้" (Debt-to-Income Ratio) โดยแปลงข้อมูลภาระหนี้สินจากแบบช่วง (Categorical) ให้เป็นตัวเลขสัดส่วนค่ากลาง (Numerical Metric) เพื่อให้โมเดลประเมินระยะห่างและความหนักหน่วงของหนี้ได้อย่างสมเหตุสมผล
4. **Risk Scoring (The Core Logic):** นิยามความเสี่ยงทางการเงินจาก "ผลลัพธ์ที่เกิดขึ้นจริง" โดยใช้ประวัติการหมุนเงินไม่ทัน (Shortage) เป็นเป้าหมายหลัก (Target Variable) เพื่อป้องกันปัญหา Target Leakage โดยแบ่งเป้าหมายเป็น 2 กลุ่ม (Binary Classification):
    - **Safe (ปลอดภัย):** กลุ่มที่ "ไม่เคย" หมุนเงินไม่ทันเลยในรอบ 1 ปี (การเงินมั่นคง)
    - **At Risk (มีความเสี่ยง):** กลุ่มที่เคยหมุนเงินไม่ทันอย่างน้อย 1 ครั้งต่อปี (ต้องเฝ้าระวัง)

### Phase 2: การวิเคราะห์ Insight และการแสดงผล (Deep EDA & Visualization)
*เป้าหมาย: ค้นหาความสัมพันธ์ที่ซ่อนอยู่ในข้อมูล เพื่อตอบโจทย์ "ใครคือกลุ่มที่น่ากังวลที่สุด"*
1. **Generation Analysis:** เปรียบเทียบภาระหนี้สินในแต่ละรุ่น (Stacked Bar Chart)
2. **Behavioral Impact:** วิเคราะห์พฤติกรรม "การซื้อของตามใจ" (Impulse Buying) สัมพันธ์กับระดับความเสี่ยงอย่างไร (Line Chart แสดงแนวโน้มค่าเฉลี่ย)
3. **Income vs. Risk Analysis:** ใช้ 100% Stacked Bar Chart วิเคราะห์สัดส่วนระดับความเสี่ยงในแต่ละช่วงรายได้ เพื่อพิสูจน์ว่ารายได้สูงไม่ได้แปลว่าความเสี่ยงต่ำเสมอไป
4. **Coping Strategy:** วิเคราะห์แนวทางรับมือวิกฤตเศรษฐกิจของแต่ละ Gen (Grouped Bar Chart)
5. **Financial Literacy vs. Risk:** ใช้ Boxplot วิเคราะห์การกระจายตัวของ "ระดับความรู้ทางการเงิน" แบ่งตาม "กลุ่มความเสี่ยง" เพื่อดูว่าความรู้เป็นภูมิคุ้มกันความเสี่ยงที่แท้จริงหรือไม่

### Phase 3: การพัฒนาและทดสอบโมเดล (Machine Learning Pipeline)
*เป้าหมาย: สร้างปัญญาประดิษฐ์ที่จดจำพฤติกรรมความเสี่ยง*
1.  **Model Selection:**
    *   **Logistic Regression:** เพื่อดูความน่าจะเป็นและน้ำหนักปัจจัยของความเสี่ยง (Interpretability)
    *   **KNN (K-Nearest Neighbors):** เพื่อจำแนกกลุ่มตามความคล้ายคลึงของพฤติกรรม (Baseline Comparison)
2.  **Training & Evaluation:**
    *   แบ่งชุดข้อมูลเป็น Training Set (80%) และ Testing Set (20%)
    *   ใช้ StandardScaler ปรับสเกลข้อมูลให้สมดุลก่อนฝึกสอน
    *   ปรับจูน Hyperparameter: Logistic Regression ใช้ C=0.001, class_weight='balanced' เพื่อให้โมเดลไม่ลำเอียงและไม่ Overfitting
    *   ฝึกสอนและประเมินประสิทธิภาพของทั้งสองโมเดลด้วย Accuracy, Classification Report และ Confusion Matrix

### Phase 4: การนำไปใช้และสรุปผล (Deployment & Recommendations)
1. **Strategic Insights:** สรุปปัจจัยที่มีผลต่อความเสี่ยงมากที่สุด (Feature Importance) โดยวิเคราะห์จากค่าน้ำหนัก (Coefficients) ของ Logistic Regression
2. **Prediction Module (Hybrid Mock-up):** พัฒนาระบบรับค่าตัวแปรจำลอง (Persona) เพื่อรันแบบจำลองและทำนายผลความเสี่ยงแบบ Real-time พร้อมแสดงระดับความมั่นใจ (Probability)
3. **Actionable Recommendations:** ให้ข้อเสนอแนะสำหรับการปรับปรุงพฤติกรรมการเงินแยกตามระดับความเสี่ยงที่ทำนายได้จริง
